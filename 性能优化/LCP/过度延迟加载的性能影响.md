### 面试预判（必考题）

这个项目写完，面试官 90% 会问这几个问题，请务必准备好：

1.  **问：你的 SKU 笛卡尔积算法是怎么写的？**
    *   *答（思路）：* “主要是利用 `reduce` 方法。假设有两个数组 `['红', '蓝']` 和 `['大', '小']`。我通过 `reduce` 遍历，把前一个结果和当前数组进行两两组合。如果规格特别多，为了防止页面卡顿，我还做了虚拟滚动或者分页处理。”（手动写一下这个算法，面试常考）。

2.  **问：权限管理中，按钮级别的权限是怎么控制的？**
    *   *答（思路）：* “我封装了一个自定义指令 `v-permission`。在指令的 `mounted` 钩子里，判断当前按钮需要的权限标识（如 `'product:add'`）是否存在于用户的权限列表中。如果不存在，直接 `el.parentNode.removeChild(el)`，从 DOM 树中移除该元素，防止用户通过控制台修改 CSS 显示按钮。”

3.  **问：Axios 封装里，Token 无感刷新是怎么做的？**
    *   *答（思路）：* “在响应拦截器里拦截 `401` 错误。如果检测到 token 过期，先将后续请求挂起（存在一个队列里），然后用 `refreshToken` 去换新 token。换成功后，重发队列里的请求；换失败则强制登出。”

### 面试预判（必问的技术点）

1.  **问：你的 SKU 笛卡尔积算法是怎么写的？**
    *   *答（思路）：* “我使用了 `reduce` 方法。初始是一个空数组 `[[]]`，然后遍历每一个规格数组，把当前结果集和新的规格数组进行双重循环组合。比如先有 `['红']`，遇到 `['S', 'M']`，就变成 `['红', 'S']` 和 `['红', 'M']`。虽然是 O(n^2) 复杂度，但考虑到规格通常不超过 3-4 层，前端计算完全没问题。”

2.  **问：按钮级别的权限控制（v-auth）是怎么实现的？**
    *   *答（思路）：* “我定义了一个自定义指令 `v-auth`。在 `mounted` 钩子中，获取当前用户的权限列表（存在 Pinia 或 LocalStorage）。判断指令传进来的权限标识（比如 `'order:delete'`）是否在列表里。如果不在，我就 `el.parentNode.removeChild(el)`，直接把在这个 DOM 元素从页面上移除，防止用户通过控制台修改 CSS 显示出来。”

3.  **问：异步加载的级联选择器，数据回显怎么做的？**
    *   *答（思路）：* “这是个难点。因为回显时，只有 ID（比如 `[11, 1101, 110101]`），但下拉框的数据还没加载回来。我的做法是：在组件初始化时，根据回显的 ID，并发请求这三级的数据，手动把数据塞进级联组件的 `options` 里，这样组件就能正确找到 label 进行显示了。”
